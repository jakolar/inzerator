# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project purpose

Inzerator generates 3D village viewers + drone-style real-estate videos from Czech RÚIAN + ČÚZK ortofoto + DSM data. It's a spin-off from the `gtaol/` 3D-reconstruction work, focused on listing/marketing tooling. MVP / personal use.

Two parallel viewer stacks coexist:

- **v1 — single-file viewer** (`hnojice_multi.html`, ~2.5k lines + `viewer-realtor-overlay.js`, ~1.9k lines): canonical reference for the realtor overlay (drone video, parcel highlights, presentation mode). Generated by `gen_multitile.py` from an OS tile grid. `hnojice_multi.html` and `hnojice_lod_multi.html` are intentionally tracked; everything else `*_multi.html` is gitignored.
- **v2 — multi-LOD viewer** (`v2.html`, `tiles_v2_<slug>/`): current pipeline. Per-slug directory holds `panorama.glb` + `details/{outer,closeup,inner}.glb` (Draco-compressed) and `heightfield/manifest.json` with LERC heightmaps + KTX2 ortho tiers. Served by `server.py` and listed in `index.html` ("Lokace").
- **Heightfield viewer** (`heightfield/index.html`, ~3.9k lines): LERC + KTX2 streaming variant of v2, opened via `/heightfield/?slug=<slug>` from the index. ~10 MB per region.
- **Image edit UI** (`image-edit/index.html`): OpenAI gpt-image-1 proxy for orthophoto / mesh-texture repair.

## Server (`server.py`, ~3.5k lines)

Single threaded `http.server.ThreadingHTTPServer` on **port 8080**, bound to `0.0.0.0`. Plain HTTP — WebCodecs / MediaRecorder / clipboard need a secure context, so wrap with Tailscale Serve (or any TLS proxy) when accessing from outside `localhost`. Self-signed cert for `jans-mac-mini.tailfe475e.ts.net` is checked in.

Force-IPv4 monkey-patch on `socket.getaddrinfo` at module load — ČÚZK's IPv6 path is dead from this LAN; v4 works. If you see `Connection refused` on ČÚZK endpoints, that shim is the first thing to check.

Concurrency caps: `_CUZK_SEM = Semaphore(6)` (across all threads) and `_OSM_SEM = Semaphore(2)` (Overpass / OSM TOS). Don't raise without a reason — 9 HD tiles × 6 workers already saturates ČÚZK.

Endpoints (all under `ProxyHandler.do_GET` / `do_POST`):

- `/proxy/ortofoto`, `/proxy/cadastre`, `/proxy/osm` — ČÚZK / OSM tile/WMS proxies, in-process LRU caches (`_CUZK_CACHE`, `_BoundedCache`).
- `/api/locations`, `/api/ruian/search?q=`, `/api/sjtsk2wgs`, `/api/jobs[/<id>][/retry|/cancel]` — Lokace UI; backed by `locations.py`.
- `/api/buildings`, `/api/parcels`, `/api/parcel-at-point`, `/api/roads`, `/api/poi`, `/api/wiki`, `/api/building-detail` — viewer queries.
- `/api/image-edit` (POST) + `/api/image-edit/prompt`, `/api/image-edit/status` — OpenAI gpt-image-1 proxy. Refuses with 503 unless `INZERATOR_API_TOKEN` is set (intentional: shared LAN, OpenAI billing attaches to the key). System prompt is read server-side from `image_edit_prompt.txt` — single source of truth, clients cannot override.

`locations.start_worker()` is called from `__main__` — kills the background job worker if you skip it.

## v2 pipeline (`locations.py`, ~1k lines)

`STEP_NAMES = ("panorama", "sm5", "outer", "closeup", "inner", "compress", "heightfield")`. Note: `tests/test_locations_unit.py` still asserts the older 6-step tuple — outdated, not a regression to chase unless you're also touching that test.

Per-step output lives under `tiles_v2_<slug>/`:

| Step | Output | Notes |
|------|--------|-------|
| `panorama` | `panorama.glb` | `gen_panorama.py` — DMR5G 30 km terrain, in-subprocess |
| `sm5` | `.sm5_ok` sentinel | resolves MAPNOM via ČÚZK `KladyMapovychListu/25`, auto-downloads `dmpok_tiff_*` (SM5 DSM) + `ortofoto_*` JPEGs into cache. In-process. |
| `outer` | `details/outer.glb` | `gen_detail.py --half 2500 --step 2.5 --fade-to panorama` |
| `closeup` | `details/closeup.glb` | `--half 1500 --step 1.5 --fade-to outer` |
| `inner` | `details/inner.glb` | `--half 500 --step 0.5 --fade-to closeup` (~4 M verts) |
| `compress` | `.compress_ok` sentinel | `draco_compress_glb.py` on each detail; originals moved to `_orig_uncompressed/` |
| `heightfield` | `heightfield/manifest.json` + LERC + KTX2 tiers | `gen_heightfield.py` — LOD streaming assets for `heightfield/index.html` |

`location_status()` reports `missing` / `partial` / `ready`. `STEP_TIMEOUT_SECS = 3600` (cold DMR5G can take 10–30 min).

`tiles_v2_<slug>/manifest.json` is the source of truth for `cx, cy, half, step, ground_z, sjtsk_bbox, wgs_bbox, details[]`. Atomic write pattern: write `.json.tmp` then `.replace()`.

## Cache layout

`cache/` is **a symlink** to `../gtaol/cache` — DSM TIFFs (`dmpok_tiff_*`) and ortofoto JPEGs (`ortofoto_*`) are shared with the gtaol experimental project. Override with `INZERATOR_CACHE` env when running from another worktree. The cache is large (regenerable from ČÚZK).

`cache/jobs/` holds per-job log JSON written by the Lokace worker.

## Heightfield generation (`gen_heightfield.py`)

LOD ring assets for the streaming viewer. Defaults: ortho tiers = `mid,high,ultra`, format = `lerc` (50% smaller than PNG, WASM-decoded), bare-earth DMR5G **cached** to `<slug>_bare.lerc` and skipped on rerun unless `--refresh-bare`. SM5 ortofoto auto-fetched via ČÚZK ATOM feed unless `--no-fetch-missing`.

```
python3 gen_heightfield.py --slug hnojice                 # full regen for one slug
python3 gen_heightfield.py --slug hnojice --refresh-bare  # also re-fetch DMR5G
python3 dispatch_heightfield.py                           # all ready locations missing heightfield/
python3 dispatch_heightfield.py --only foo,bar --force    # subset, regenerate existing
python3 refresh_ortho.py --tiers super                    # opt-in 16384² super tier (~40 MB KTX2, 128 MB GPU)
python3 refresh_cadastre.py --size 8192 --only foo        # re-pull cadastre PNGs only (server :8080 must be up)
```

`refresh_cadastre.py` and `refresh_ortho.py` both expect `tiles_v2_<slug>/heightfield/manifest.json` to exist (they update rings, not generate from scratch).

## Common commands

```bash
# Install pinned deps (Python 3.9.6 / macOS).
pip install -r requirements.txt

# Server (port 8080, 0.0.0.0). Loads .env first (OPENAI_API_KEY, INZERATOR_API_TOKEN).
set -a; source .env; set +a; python3 server.py

# Tests — pytest. Unit tests run standalone; *_api.py and test_parcels_endpoint.py
# need server.py running on :8080.
pytest                                            # all
pytest tests/test_locations_unit.py               # unit only (no server needed)
pytest tests/test_parcels_endpoint.py -k area     # single test
```

Generating one v2 location end-to-end is normally driven from the Lokace UI (`index.html` → typeahead RUIAN search → "+ Nová lokace"). Manual fallback:

```bash
python3 gen_panorama.py --region <slug> --center-sjtsk=<cx>,<cy>
python3 gen_detail.py   --region <slug> --slug outer   --center-sjtsk=<cx>,<cy> --half 2500 --step 2.5 --fade-to panorama
python3 gen_detail.py   --region <slug> --slug closeup --center-sjtsk=<cx>,<cy> --half 1500 --step 1.5 --fade-to outer
python3 gen_detail.py   --region <slug> --slug inner   --center-sjtsk=<cx>,<cy> --half 500  --step 0.5 --fade-to closeup
python3 draco_compress_glb.py tiles_v2_<slug>/details/inner.glb tiles_v2_<slug>/details/inner.glb
python3 gen_heightfield.py --slug <slug>
```

Negative S-JTSK coords need the `=` form (`--center-sjtsk=-547700,-1107700`), otherwise argparse swallows the minus as a flag.

## Conventions specific to this repo

- **Negative ČÚZK coordinates** are normal — S-JTSK Krovak East-North has negative X and Y across all of Czechia. Always use `--center-sjtsk=cx,cy` form on CLI scripts.
- **Atomic manifest writes**: `tmp = path.with_suffix('.json.tmp'); tmp.write_text(...); tmp.replace(path)`. Match it when adding new manifest writers.
- **Slug rules**: lowercase, ASCII, hyphen-separated. `locations.slugify` does NFD-fold; `slugifyJS` in `index.html` must stay byte-for-byte equivalent (the UI auto-fills slug from the chosen RUIAN address; mismatch → wrong directory).
- **Sentinels over GLBs**: `.sm5_ok` and `.compress_ok` are zero-byte markers used by `expected_glb()` / `location_status()` so the resume-skip loop can `.exists()`-check uniformly. Don't replace with GLB-existence checks — those steps don't produce a single artifact.
- **Cache-bust on `.glb`**: v2.html uses `BUST = Date.now()` because the same filename gets rewritten by `compress` and `refresh_*` scripts. Preserve that when editing the loader.
- **Czech UI, English code/commits** — matches the global rule in `~/.claude/CLAUDE.md`.
